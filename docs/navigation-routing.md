# Navigation Property Routing

OhData supports `GET /Parents(1)/Children` style routes per the OData spec. These are distinct from `$expand` (which embeds related data in the parent response) — navigation routes return the related collection as a top-level response.

## Declaring a navigation route

Pass a handler delegate to `HasMany`, `HasOptional`, or `HasRequired`:

```csharp
public class OrderProfile : EntitySetProfile<Guid, Order>
{
    public OrderProfile(AppDbContext db) : base(x => x.Id)
    {
        // Declares the nav property in the EDM model AND registers a route
        HasMany(x => x.Lines,
            getAll: (orderId, ct) =>
                Task.FromResult<IEnumerable<OrderLine>>(
                    db.OrderLines.Where(l => l.OrderId == orderId).ToList()));

        GetQueryable = (_) => Task.FromResult(db.Orders.AsQueryable());
    }
}
```

This registers: `GET /odata/Orders({key})/Lines`

## Route behavior

- If the parent key doesn't exist (handler returns `null`), the framework returns `404 NotFound`.
- Auth from the parent profile's `RequireAuthorization()` / `RequireRoles()` is applied to navigation routes.
- `HasOptional` / `HasRequired` with a `get` delegate registers `GET /Parents({key})/{PropertyName}` for a single related entity.

## Without a handler

Calling `HasMany`, `HasOptional`, or `HasRequired` without a handler only declares the navigation property in the EDM model — used for `$expand` and `$metadata`. No HTTP route is registered.

```csharp
// Declares nav property in EDM for $expand; no route registered
HasMany(x => x.Lines);

// Both: EDM declaration + route
HasMany(x => x.Lines, getAll: (orderId, ct) => ...);
```

## `$expand`

`$expand` works automatically on the `GetQueryable` path when navigation properties are declared and `ExpandEnabled = true`. EF Core's `Include()` is generated by `ODataQueryOptions.ApplyTo(IQueryable)` when the client sends `$expand=Lines`.

Navigation property routing and `$expand` are complementary: routing returns the related collection as the top-level response, while `$expand` embeds it inside the parent entity response.
